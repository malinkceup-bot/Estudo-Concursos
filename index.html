━━━ PARTE 4: TRAVAR SEU SUPABASE (recomendado) ━━━
7 Impedir que outra pessoa crie conta no seu Supabase
⚠️ Faça isso SOMENTE depois de já ter clicado em "Finalizar Configuração e Entrar" acima e conseguido logar com sucesso.

📌 Volte no SQL Editor do seu Supabase → New query
📌 No SQL abaixo, troque SEU-EMAIL-AQUI@gmail.com pelo e-mail que você acabou de usar no Passo 6
📌 Cole e clique em Run ▶️

Isso garante que, mesmo que a URL e a chave do seu Supabase vazem ou sejam descobertas por alguém, essa pessoa não consiga criar conta nova ali dentro — só o seu e-mail é aceito. 
create or replace function restringir_cadastro_email()
returns trigger
language plpgsql
security definer
set search_path = public
as $$
begin
  if lower(new.email) <> lower('SEU-EMAIL-AQUI@gmail.com') then
    raise exception 'Cadastro não permitido neste projeto. Este Supabase já pertence a outra pessoa.';
  end if;
  return new;
end;
$$;
drop trigger if exists restringir_cadastro_trigger on auth.users;
create trigger restringir_cadastro_trigger
before insert on auth.users
for each row execute function restringir_cadastro_email();
